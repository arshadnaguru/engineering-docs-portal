# Reference

Reference documentation is **information-oriented** — it describes the system and its components. Think of it as a technical dictionary you consult when you need exact specifications.

## Available References

- [Markdown Style Guide](markdown-style-guide.md) — Formatting standards for all documentation
- [API Documentation Template](api-doc-template.md) — Standard template for documenting REST API endpoints
- [Project Folder Structure](folder-structure.md) — Standard directory layout for documentation repositories

!!! info "Reference vs How-To"
    **Reference** documents describe _what things are_. **How-to guides** describe _how to do things_. A reference page for Git commands lists them with descriptions. A how-to guide walks you through using Git for a specific task.
