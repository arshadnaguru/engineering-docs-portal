# How-To Guides

How-to guides are **task-oriented** instructions that help you accomplish a specific goal. Unlike tutorials, they assume you already have foundational knowledge and need practical steps to get something done.

## Available Guides

- [Deploy a Dockerized Application on Linux](docker-deploy-linux.md) — Production deployment with Docker Compose
- [Format Raw Notes into Structured Markdown](format-notes-markdown.md) — Use our automation script to convert messy notes
- [Set Up a Documentation Review Workflow](doc-review-workflow.md) — Configure branch-based review for your docs team

!!! info "How-to vs Tutorial"
    **Tutorials** teach by guiding you through a complete learning experience. **How-to guides** solve a specific problem you already understand. Think: _tutorial = cooking class_, _how-to = recipe_.
