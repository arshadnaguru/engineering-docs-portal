# Explanation

Explanation documentation is **understanding-oriented** — it provides context, background, and reasoning. These articles help you understand _why_ we do things a certain way, not _how_ to do them.

## Available Articles

- [Why Diátaxis?](why-diataxis.md) — The documentation framework behind our organization strategy
- [Docs-as-Code Philosophy](docs-as-code.md) — Why we treat documentation like software
- [Information Architecture Principles](information-architecture.md) — How we decide where information lives

!!! note "Explanation vs Reference"
    **Explanation** discusses concepts, context, and reasoning. **Reference** provides precise technical facts. An explanation of REST APIs would cover why they exist and design principles. A reference for REST APIs would list endpoints, parameters, and response formats.
