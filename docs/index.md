# Engineering Documentation Portal

Welcome to the **Engineering Documentation Portal** — a centralized, version-controlled knowledge base organized using the [Diátaxis documentation framework](https://diataxis.fr/).

## Purpose

In fast-moving engineering environments with high team turnover, institutional knowledge is constantly at risk of being lost. This portal solves that by:

- **Standardizing** all documentation into strict Markdown format
- **Organizing** content using the Diátaxis framework (Tutorials, How-To Guides, Reference, Explanation)
- **Automating** the conversion of raw notes and legacy docs into structured guides
- **Version-controlling** everything through Git-based workflows

## How This Portal Is Organized

| Section | Purpose | Example |
|---------|---------|---------|
| **Tutorials** | Learning-oriented, step-by-step walkthroughs | _Setting Up a Python ML Environment_ |
| **How-To Guides** | Task-oriented, practical instructions | _Deploy a Dockerized App on Linux_ |
| **Reference** | Information-oriented, technical descriptions | _Markdown Style Guide_ |
| **Explanation** | Understanding-oriented, conceptual discussions | _Why Diátaxis?_ |

## Quick Start

New to this portal? Start here:

1. Read the [Markdown Style Guide](reference/markdown-style-guide.md) to understand formatting standards
2. Review [Why Diátaxis?](explanation/why-diataxis.md) to understand our organizational philosophy
3. Check the [Contributing Guidelines](https://github.com/arshadnaguru/engineering-docs-portal/blob/main/CONTRIBUTING.md) before submitting documentation
4. Use the [formatting script](how-to/format-notes-markdown.md) to convert raw notes into structured Markdown
